# 전문가 퍼소나 인터뷰 랩

전문 지식, 가치관, 이해관계, 판단 규칙을 가진 가상 전문가 퍼소나를 만들고 인터뷰하는 교육용 웹 앱입니다. 퍼소나는 실제 전문가나 실제 조사 결과를 대체하지 않으며, 특정 관점에서 어떤 쟁점과 검증 필요 항목이 드러나는지 탐색하도록 설계되어 있습니다.

## 주요 기능

- 전문가 퍼소나 카드 생성: 역할, 전문 영역, 지식, 가치/이해관계, 판단 규칙, 말하기 한계 입력
- 공동 대화장: 여러 퍼소나가 패널처럼 한 번에 발언
- 개별 대화창: 특정 퍼소나와 1:1 심층면접
- 언어 설정: 시스템 언어, 전문가 응답 언어, 보고서 언어를 별도로 선택
- 회의 유형: 전문가 좌담회, 심층면접, 델파이 라이트, 포커스 그룹, 이해관계자 테스트, 광고/서비스 반응, 분류 타당성 감사
- 비용 선택: 경제형, 균형형, 심화형
- 크레딧 제한: 사용자별 예산, 라운드 수, 출력 토큰 수, 퍼소나 수 제한
- 요약 및 파일 내보내기: Markdown, Word `.docx`, 한글 `.hwpx`
- GPT 에디터: 회의록을 제출용 보고서 초안으로 다듬어 `편집본`으로 저장
- Railway 배포와 Postgres 영구 저장 지원

## 로컬 실행

```bash
npm install
cp .env.example .env
npm start
```

브라우저에서 `http://localhost:3000`을 엽니다.

`OPENAI_API_KEY`가 없으면 모의 응답으로 실행됩니다. 모의 응답은 크레딧을 차감하지 않습니다.

## 크레딧 정책

이 앱은 학생별 사용량을 credits로 제한한다.

- 구매 기준: 1 credit = 1원
- API 원가 기준: 원가 1 USD = 14,000 credits 차감
- 학생 기본 한도: 30,000 credits

예를 들어 OpenAI API 사용으로 원가 $0.10이 발생하면 다음과 같이 차감한다.

$0.10 × 14,000 = 1,400 credits

학생에게 30,000 credits를 부여하면 API 원가 기준으로 약 $2.14까지 사용할 수 있다.
실제 환율을 약 1 USD = 1,400원으로 보면 API 원가 약 3,000원 정도에 해당한다.

주의:
크레딧 차감 공식은 실제 환율 계산이 아니라 수업 운영용 내부 과금/한도 계산 공식이다.

## 추천 운영 방식

- 퍼소나는 3~4명으로 시작합니다.
- 공동 대화장을 기본으로 쓰고, 꼭 필요한 경우만 개별 심층면접을 사용합니다.
- 학생에게 경제형/균형형/심화형의 비용 차이를 안내합니다.
- 회의가 길어지면 요약을 만든 뒤 새 세션으로 이어갑니다.
- 실제 조사, 실제 전문가 자문, 실제 소비자 테스트처럼 표현하지 않도록 지도합니다.

## Railway 배포

1. GitHub 저장소에 코드를 올립니다.
2. Railway에서 GitHub repo를 배포합니다.
3. Variables에 `.env.example`의 값을 설정합니다.
4. 기록을 유지하려면 Railway Postgres를 추가하고 `DATABASE_URL`을 연결합니다.
5. Railway Workspace Usage에서 별도 하드 리밋을 설정해 인프라 비용도 제한합니다.

## 교육적 주의

이 앱의 출력은 시뮬레이션입니다. 전문가 퍼소나는 주어진 카드와 대화 맥락을 바탕으로 관점을 유지하도록 설계되었지만, 실제 전문가 의견이나 실제 집단의 반응을 보장하지 않습니다. 보고서에는 "가상 퍼소나 인터뷰 결과"임을 명시하고, 중요한 주장에는 검증 필요 항목을 남겨야 합니다.

## 회의 종류

회의 종류는 복잡한 별도 엔진이 아니라 운영 프롬프트와 응답 출력 형식을 바꾸는 선택지입니다.

- 전문가 좌담회: 퍼소나별 핵심 의견, 합의점, 불일치점, 검증 필요 주장, 회의록 기록용 결론
- 전문가 면접: 퍼소나의 핵심 답변, 사용한 지식, 판단 기준, 한계, 추가 질문
- 델파이 라이트: 퍼소나별 판단, 점수 또는 등급, 불일치 이유, 합의 가능 조건, 최종 유보 사항. 실제 델파이 조사가 아니라 “델파이 라이트 시뮬레이션”으로 표시합니다.
- 포커스 집단 인터뷰: 첫인상, 공통 반응, 갈리는 반응, 불신 요인, 개선 제안. 실제 소비자 조사 결과처럼 표현하지 않습니다.
- 이해관계자 반응 테스트: 이해관계자별 반응, 이익, 우려, 갈등 지점, 수용 조건, 설명 방식 개선안
- 광고·소비자 반응 테스트: 매력 요소, 거부감, 신뢰감, 기억에 남는 표현, 오해 가능성, 수정 제안. 실제 시장조사 결과처럼 표현하지 않습니다.
- 분류 타당성 감사: 정확도, 신뢰도, 진실성, 타당도, 사회적 정합성, 이의제기 가능성, 검증 가능한 주장, 검증 불가능한 추정, 가치 판단, 수정된 분류안

## 퍼소나 생성

퍼소나는 단순 캐릭터가 아니라 특정 지식, 경험, 가치, 판단 규칙을 가진 가상 관점 모델입니다. 생성 항목은 이름, 역할/사회적 위치, 전문성/대표 경험, 사용할 지식, 중시 가치, 판단 규칙, 말하기 방식, 한계 또는 금지할 말을 포함합니다.

퍼소나 응답 프롬프트는 실제 인간 전문가를 대체한다고 말하지 않기, 실제 소비자 조사 결과처럼 말하지 않기, 자료 없이 사실을 만들지 않기, 사실·추정·가치 판단 구분하기, 특정 집단을 고정관념으로 단정하지 않기, 민감한 개인정보를 요구하지 않기, 퍼소나의 지식과 한계 안에서 답하기, 모르는 것은 모른다거나 검증 필요로 표시하기를 강제합니다.

## 맥락 유지

서버는 전체 회의록을 저장하지만, 모델 호출에는 회의 기본 정보, 퍼소나 카드, rolling summary, 최근 회의록 일부, 학생의 새 입력만 보냅니다. 요약본 생성 버튼을 누르면 `session.rolling_summary`가 갱신되고, 이후 공동 대화장과 퍼소나별 개별 대화창 호출에 이 요약이 포함됩니다. 회의가 길어지면 화면에 “요약본 생성을 눌러 맥락을 압축하세요” 안내가 표시됩니다.

## 요약본

요약본은 학생 제출용 구조로 생성됩니다. 반드시 핵심 논의 요약, 퍼소나별 주요 주장, 합의점, 불일치점, 검증 가능한 사실, 확인 불가능한 추정, 가치 판단, 검증 필요 주장, 인간이 최종 판단해야 할 부분, 학생 성찰 질문을 포함합니다. 생성된 요약 메시지는 회의록에 저장되고 화면에 표시되며, 동시에 `session.rolling_summary`를 갱신합니다.
