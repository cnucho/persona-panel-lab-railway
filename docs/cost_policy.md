# 크레딧 정책

## 기본 환산

- 학생별 예산: 30,000 credits
- 10 credits = 약 1원
- 30,000 credits = 약 3,000원

환경변수:

```env
CREDIT_BUDGET_PER_USER=30000
KRW_PER_CREDIT=0.1
USD_TO_KRW=1400
```

## 차감 공식

```text
예상 달러 비용 = 입력토큰/1,000,000 × 입력단가 + 출력토큰/1,000,000 × 출력단가
예상 원화 비용 = 예상 달러 비용 × USD_TO_KRW
차감 크레딧 = ceil(예상 원화 비용 / KRW_PER_CREDIT)
```

## 모델 단가 테이블

앱 코드의 `MODEL_PRICES`를 수정하면 됩니다. 기본값은 다음 계열을 포함합니다.

- gpt-5.5
- gpt-5.5-pro
- gpt-5.4
- gpt-5.4-mini
- gpt-5.4-nano
- gpt-5.4-pro

## 비용을 줄이는 운영 방식

- 기본 모델은 mini 사용
- 요약은 nano 사용
- 퍼소나 3~4명 권장
- 공동 대화장은 1회 호출로 여러 퍼소나 응답을 생성
- 퍼소나별 개별 대화창은 꼭 필요할 때만 사용
- 요약본 생성은 마지막 1회만 사용
- 전체 회의록을 매번 보내지 않고 최근 메시지와 요약만 사용
