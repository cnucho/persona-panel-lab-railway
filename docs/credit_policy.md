# 크레딧 정책

크레딧은 수업 단위의 사용량 제한 장치입니다. 실제 API 비용을 원화로 추정한 뒤 `KRW_PER_CREDIT` 값으로 나누어 차감합니다.

## 기본 원칙

- 사용자 1명 기본 한도: `30,000 credits`
- 원화 환산: `1 credit = 0.1 KRW`
- 모의 응답 모드: 차감 `0 credits`
- 실제 API 호출: OpenAI usage 값 기준 차감

## 서버 제한

```env
CREDIT_BUDGET_PER_USER=30000
MAX_PERSONAS=5
MAX_ROUNDS_PER_SESSION=8
MAX_MESSAGES_PER_SESSION=100
MAX_OUTPUT_TOKENS=700
```

이 제한은 앱 화면 선택보다 우선합니다. 사용자가 심화형을 선택해도 서버 예산이나 라운드 제한을 넘으면 요청이 거부됩니다.

## 깊이별 운영

- 경제형: 짧은 답변, 낮은 비용, 반복 질문에 적합
- 균형형: 기본 수업 활동에 적합
- 심화형: 전문 지식, 가치 충돌, 반론, 검증 필요 항목을 더 깊게 다룸

## 교사용 권장값

- 일반 수업: `CREDIT_BUDGET_PER_USER=10000`부터 시작
- 프로젝트 수업: `30000`
- 긴 보고서 활동: `50000` 이상, 단 Railway 자체 사용량 제한도 함께 설정

학생에게는 심화형이 더 많은 비용을 사용한다는 점을 안내하고, 모든 라운드를 심화형으로 진행하지 않도록 운영하는 것이 좋습니다.
